Tutorial de VoiceCalendar

Aprende a configurar VoiceCalendar en pocos pasos para programar eventos con tu voz. ¡Es fácil y rápido, incluso si no sabes de tecnología!

Cómo configurar VoiceCalendar

Paso 1: Obtén tu chat_id de Telegram

VoiceCalendar envía confirmaciones y recordatorios a través de Telegram usando mi bot. Necesitas un chat_id para recibir los mensajes.

  1. Abre la app de Telegram en tu teléfono o computadora.
  2. En la barra de búsqueda, escribe @myidbot y selecciona el bot @myidbot.
  3. Escribe /start y presiona enviar. Luego escribe /getid y presiona enviar.
  4. El bot te dará un número, como 123456789. Esto es tu chat_id. Cópialo (puedes escribirlo en una nota o en tu teléfono) porque lo necesitarás más adelante.
  5. Nota: Si no ves el mensaje del bot, asegúrate de estar en un chat privado con @myidbot. No compartas tu chat_id con nadie.

Paso 2: Crea una cuenta en AssemblyAI

AssemblyAI convierte tu voz en texto para que VoiceCalendar entienda tus comandos. Necesitas una API key de AssemblyAI.

  1. Ve a www.assemblyai.com en tu navegador.
  2. Haz clic en Sign Up (arriba a la derecha) y crea una cuenta gratis con tu correo electrónico y una contraseña. También puedes usar tu cuenta de Google.
  3. Una vez registrado, inicia sesión. Verás una pantalla con un panel (llamado "Dashboard").
  4. En el Dashboard, busca una sección llamada API Key o Your API Token. Verás un código largo, como abc123xyz.
  5. Copia ese código (tu API key) y guárdalo en una nota o en tu teléfono. Lo usarás en el siguiente paso.
  6. Nota: No compartas tu API key con nadie. Si no ves la API key, revisa la sección de "Settings" o "Account" en AssemblyAI.

Paso 3: Importa la plantilla en Make.com

Make.com conecta VoiceCalendar con AssemblyAI, Google Calendar, Telegram y un Data Store. Usa mi plantilla para configurar todo rápidamente.

  1. Ve a www.make.com y crea una cuenta gratis con tu correo electrónico o inicia sesión si ya tienes una.
  2. En la pantalla principal de Make.com, haz clic en Create a new scenario (botón azul, arriba a la izquierda).
  3. Descarga mi plantilla de Make. Esto descargará un archivo llamado voicecalendar_blueprint.json.
  4. En Make.com, haz clic en los tres puntos (...) en la esquina inferior izquierda y selecciona Import Blueprint.
  5. Sube el archivo voicecalendar_blueprint.json que descargaste (busca el archivo en tu computadora o teléfono).
  6. Verás una pantalla con varios módulos (círculos conectados por líneas). Estos módulos incluyen Webhook, AssemblyAI, Google Calendar, Telegram y Data Store.
  7. Nota: No actives el escenario todavía (el botón de encendido debe estar gris). Configura el webhook y los módulos primero en los siguientes pasos.

Paso 4: Configura el webhook en Make.com

La plantilla incluye un módulo de webhook que recibe los audios de VoiceCalendar. Necesitas crear la URL del webhook.

  1. En el escenario de Make.com (que creaste en el Paso 3), haz clic en el módulo Webhook (es el primer círculo, normalmente con un ícono de un gancho).
  2. Haz clic en Create a webhook o en Add si ya existe un webhook.
  3. Make.com generará una URL que empieza con https://hook.eu2.make.com/ seguido de un código, como https://hook.eu2.make.com/xyz123.
  4. Copia esta URL (haz clic en el botón "Copy address to clipboard") y guárdala en una nota, porque la usarás más adelante.
  5. Haz clic en OK para guardar el webhook.
  6. Nota: Si ves un error como "Webhook no determinado", haz clic en "Redetermine data structure" y prueba enviar un audio desde VoiceCalendar (Paso 6) para que Make.com detecte el formato.

Paso 5: Configura las conexiones de todos los módulos

La plantilla tiene varios módulos (Webhook, AssemblyAI, Google Calendar, Telegram, Data Store). Cada uno necesita una conexión específica. Configura todos los módulos como se indica a continuación:

  1. Webhook:
    • Ya configuraste el webhook en el Paso 4. Asegúrate de que la URL esté copiada y que el módulo diga "Successfully determined" o "Estructura de datos determinada".
  2. AssemblyAI:
    • Busca todos los módulos con el logo de AssemblyAI (puede haber más de uno, como "Transcribe Audio" o "Get Transcription").
    • Para cada módulo de AssemblyAI, haz clic en él y selecciona Add en el campo "Connection".
    • Pega la API key que copiaste en el Paso 2 en el campo "API Token".
    • Haz clic en Save o OK para cada módulo. Si hay varios módulos de AssemblyAI, repite este proceso para cada uno.
    • Nota: Si ves un error como "Invalid API Token", verifica que la API key sea correcta y no tenga espacios.
  3. Google Calendar:
    • Busca todos los módulos con el logo de Google Calendar (como "Create an Event").
    • Para cada módulo, haz clic en él y selecciona Add en el campo "Connection".
    • Inicia sesión con tu cuenta de Google y permite que Make.com acceda a tu calendario.
    • Elige el calendario donde quieres crear los eventos (por ejemplo, tu calendario personal).
    • Haz clic en OK para guardar. Repite para cada módulo de Google Calendar si hay más de uno.
    • Nota: Si no ves tu calendario, revisa que hayas iniciado sesión con la cuenta correcta.
  4. Telegram:
    • Busca todos los módulos con el logo de Telegram (como "Send a Message").
    • Para cada módulo, haz clic en él y selecciona Add en el campo "Connection".
    • En el campo Chat ID, pega el chat_id que obtuviste en el Paso 1.
    • No necesitas configurar un bot, ya que usarás mi bot de Telegram. Solo asegúrate de que el Chat ID sea correcto.
    • Haz clic en OK para guardar. Repite para cada módulo de Telegram si hay más de uno.
    • Nota: Si no recibes mensajes en Telegram, verifica que el chat_id sea correcto con @myidbot.
  5. Data Store:
    • Busca todos los módulos con el logo de Data Store (como "Add/Replace a Record").
    • Si no hay un Data Store creado, sigue el Paso 6 para crearlo.
    • Para cada módulo de Data Store, haz clic en él y selecciona el Data Store que crearás en el Paso 6 (por ejemplo, VoiceCalendarEvents) en el campo "Data Store".
    • Asegúrate de que los campos (Evento, Día, Fecha, Hora) estén conectados a los datos del módulo anterior (normalmente AssemblyAI). Si no están conectados, arrastra los campos desde el módulo anterior al Data Store.
    • Haz clic en OK para guardar. Repite para cada módulo de Data Store si hay más de uno.
    • Nota: Si los datos no se guardan, revisa que los campos estén correctamente conectados.
  6. Haz clic en el botón de encendido (arriba a la izquierda) para activar el escenario.
  7. Nota: Si ves un error, revisa que todas las conexiones estén completas. Puedes hacer clic en "Run once" para probar el escenario.

Paso 6: Crea un Data Store en Make.com

El Data Store guarda los detalles de tus eventos para que puedas revisarlos más tarde.

  1. En Make.com, en la pantalla principal, haz clic en Data Stores en el menú de la izquierda (puede estar en "More" si usas un teléfono).
  2. Haz clic en Create a new Data Store.
  3. En el campo "Name", escribe VoiceCalendarEvents (o cualquier nombre que prefieras, pero recuerda este nombre).
  4. Haz clic en Add field y crea al menos estos campos:
    • Nombre del campo: Evento, Tipo: Text
    • Nombre del campo: Día, Tipo: Text
    • Nombre del campo: Fecha, Tipo: Text
    • Nombre del campo: Hora, Tipo: Text
  5. Haz clic en Save para crear el Data Store.
  6. Vuelve al escenario de la plantilla (Paso 5). En cada módulo de "Data Store - Add/Replace a Record", selecciona el Data Store que acabas de crear (VoiceCalendarEvents) en el campo "Data Store".
  7. Asegúrate de que los campos (Evento, Día, Fecha, Hora) estén conectados a los datos del módulo anterior (normalmente el módulo de AssemblyAI). Si no están conectados, arrastra los campos desde el módulo anterior al Data Store.
  8. Nota: Si no sabes cómo conectar los campos, haz clic en "Run once" y revisa si el Data Store guarda los datos correctamente después de grabar un audio.

Paso 7: Conecta VoiceCalendar

Ahora que tienes todo configurado, conecta la URL del webhook a VoiceCalendar.

  1. Ve a VoiceCalendar (o abre la página principal en tu navegador).
  2. En la sección Configuración, pega la URL del webhook (del Paso 4) en el campo URL del Webhook. Asegúrate de que sea exactamente la misma URL, sin espacios.
  3. Haz clic en el botón del micrófono y di un comando de voz (ver el siguiente paso).
  4. Nota: Si ves un mensaje de error, verifica que la URL sea correcta y que el escenario en Make.com esté activado.

Paso 8: Prueba con un comando de voz

VoiceCalendar reconoce comandos en el formato exacto: Evento: [nombre], Día: [día de la semana], Fecha: [día de mes de texto], Hora: [hora en punto, es decir, 12,14,1,8]. Debes decir las palabras Evento:, Día:, Fecha: y Hora: exactamente, y los valores deben seguir el formato del ejemplo. Por ejemplo: "Evento: Cita médica, Día: viernes, Fecha: 8 de agosto, Hora: 10".

Importante: No uses formatos como "25 julio 2025", "11:45" o "mañana". Por ahora, VoiceCalendar solo entiende el formato exacto del ejemplo.

Ejemplo:
"Evento: Reunión con Ana, Día: jueves, Fecha: 25 de julio, Hora: 11"
  1. Abre VoiceCalendar y haz clic en el botón del micrófono (el círculo verde).
  2. Di tu comando de voz claramente, usando el formato exacto del ejemplo.
  3. Espera unos segundos mientras se procesa.
  4. Verifica en tu Google Calendar que el evento se creó (por ejemplo, "Cita médica" el 8 de agosto a las 10:00).
  5. Revisa Telegram para confirmar que recibiste un mensaje de mi bot con los detalles del evento.
  6. Abre Make.com, ve a Data Stores, selecciona VoiceCalendarEvents, y verifica que los detalles del evento (Evento, Día, Fecha, Hora) estén guardados.
  7. Nota: Si el evento no aparece, revisa las "Notas finales" para solucionar problemas.

Notas finales: Solución de problemas

Si algo no funciona, revisa estas soluciones:

  • Micrófono: Asegúrate de que VoiceCalendar tenga permisos para usar el micrófono. En tu teléfono, ve a Configuración > Navegador > Micrófono y permite el acceso. En computadoras, haz clic en el ícono del micrófono en la barra de direcciones.
  • Webhook: Verifica que la URL del webhook sea correcta (cópiala y pégala sin espacios). En Make.com, asegúrate de que el escenario esté activado (botón de encendido en azul).
  • AssemblyAI: Si el audio no se convierte en texto, revisa que la API key sea correcta en todos los módulos de AssemblyAI. Haz clic en cada módulo y verifica que el campo "API Token" no esté vacío.
  • Google Calendar: Asegúrate de haber seleccionado el calendario correcto en todos los módulos de Google Calendar. Si no ves el evento, revisa tu calendario en calendar.google.com.
  • Telegram: Si no recibes mensajes, verifica que el chat_id sea correcto en todos los módulos de Telegram. Abre Telegram, escribe a @myidbot, y confirma que el chat_id coincide.
  • Data Store: Si los eventos no se guardan, revisa que el Data Store VoiceCalendarEvents esté seleccionado en todos los módulos de Data Store y que los campos estén conectados.
  • Formato de voz: Si el evento no se crea, asegúrate de usar el formato exacto: "Evento: [nombre], Día: [día], Fecha: [día de mes], Hora: [hora en punto]". Por ejemplo, di "Evento: Cita médica, Día: viernes, Fecha: 8 de agosto, Hora: 10 de la mañana". Evita formatos como "25 julio 2025" o "11:45".
  • Error general: En Make.com, haz clic en "Run once" para probar el escenario. Si ves un error, revisa los módulos uno por uno para encontrar el problema.
